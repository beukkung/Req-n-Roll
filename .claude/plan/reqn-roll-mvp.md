## Requirement Summary

Internal capability-building platform for BA / PM / PO communities. Bilingual (Thai body + English brand terms). Free internal beta for 20–30 users.

**MVP scope (locked):**
1. **Home** — hero, feature overview, CTAs
2. **Skill Amp** — 30–40 question skill assessment across 6 dimensions → persona result
3. **Daily Req** — 10 starter short-form questions, instant feedback
4. **Req Gym** — 30 IIBA-*aligned* (NOT "official") practice questions across BABOK knowledge areas
5. **Setlist Templates** — 5 downloadable templates
6. **Feedback form** — rating + "used for real work?" flag + comment

**Deferred to Phase 2:** Roll Play, Req Doctor, Tracklist, Jam Room, badges, leaderboard, coaching/office hour.

## Technical Solution

### Stack
- **Next.js 14 (App Router)** + React 18 + **TypeScript**
- **Tailwind CSS** + **shadcn/ui** (Radix primitives) for accessible quiz / radio / dialog components
- **Supabase**: Postgres · Auth (magic-link or SSO) · Storage (template files) · Row-Level-Security
- **recharts** — Skill Amp radar chart (6 skills)
- **Thai font**: IBM Plex Sans Thai (modern) — English display font for headings
- Deploy: Vercel or internal host

### Data Model (Supabase Postgres)
- `skills` — 6 rows: `requirement_thinking`, `stakeholder_communication`, `process_system_thinking`, `product_value_thinking`, `agile_delivery`, `soft_skill_facilitation`
- `personas` — `clarifier`, `shipper`, `facilitator`, `value_hunter`, `scope_slayer` (slug, name_th, description_th)
- `skill_amp_questions` (skill_id, text_th, order, weight)
- `skill_amp_results` (user_id, scores jsonb, persona_slug, created_at) → **Skill Amp completion metric**
- `daily_req_questions` (date, type, prompt_th, options jsonb, answer, explanation) — type ∈ {requirement_quality, user_story_gap, stakeholder_followup, ac_testable, scope_risk, priority}
- `daily_req_completions` (user_id, date, score) → **return-rate metric**
- `req_gym_questions` (knowledge_area, difficulty, prompt_th, options, answer, explanation)
- `req_gym_attempts` (user_id, score, completed_at)
- `templates` (slug, name, category, description_th, file_path, download_count)
- `template_downloads` (template_id, user_id, at) → **download metric**
- `feedback` (user_id, rating, used_for_real_work bool, category, comment, created_at) → **feedback metric**

**RLS:** internal-only via email-domain allowlist (`@yourcompany.com`) or allowlist table.

### Persona Assignment Rule (needs user sign-off)
Top skill → persona: Clarifier←Requirement Thinking · Shipper←Agile & Delivery · Facilitator←Soft Skill/Facilitation · Value Hunter←Product/Value Thinking · Scope Slayer←Process & System Thinking. Ties broken by 2nd-highest skill.

### Routes (App Router)
```
app/
  layout.tsx                      # Nav (Home/Skill Amp/Daily Req/Req Gym/Templates) + Footer
  page.tsx                        # Home: hero + 6 feature cards + CTAs + landing copy
  skill-amp/
    page.tsx                      # intro + Start
    quiz/page.tsx                 # 30–40 Q stepper (client component)
    result/page.tsx               # persona badge + 6-skill radar + recommendations
  daily-req/page.tsx              # today's 3–5 Q, instant feedback per question
  req-gym/
    page.tsx                      # knowledge-area picker (ECBA / BABOK concept / mock)
    practice/[setId]/page.tsx     # reuses QuizStepper
  templates/page.tsx              # 5 template cards + download
  feedback/page.tsx               # rating + "used for real work?" + comment
```

### Reusable Components
`<Nav>` `<Footer>` `<Hero>` `<FeatureCard>` `<QuizStepper>` (Skill Amp + Req Gym) `<QuestionCard>` (a11y radio/checkbox) `<PersonaResult>` (radar + badge) `<DailyReqCard>` `<TemplateCard>` `<FeedbackForm>`

### Tone / Visual
High-contrast, energetic, "rock/practical" — not HR portal. Copy verbatim from spec: *"Rock the Requirement. Ship the Value."*, *"หยุดรับ requirement แบบงง ๆ แล้วเริ่มถามแบบคนคุมเกม"*, etc.

### Success Metrics Instrumentation
All metrics from spec are captured by the tables above (completions, return rate, downloads, attempts, "used for real work" flag). Internal dashboard page deferred — schema already supports it.

## Implementation Steps

1. **Scaffold** — `create-next-app` (App Router, TS, Tailwind), install shadcn/ui, recharts, `@supabase/supabase-js`, `@supabase/ssr`. Set up env (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`). → deliverable: running dev server
2. **Design system** — fonts (IBM Plex Sans Thai), color tokens, `Nav`, `Footer`, root `layout.tsx`, `Hero`, `FeatureCard`. → deliverable: Home shell renders
3. **Supabase project** — create project, write SQL migration (all tables + RLS policies + allowlist), seed the 6 `skills` + 5 `personas`, set up Storage bucket for templates. → deliverable: migration applied, RLS verified
4. **Auth** — magic-link or SSO; enforce internal allowlist at sign-up. → deliverable: only `@yourcompany.com` (or allowlist) can sign in
5. **Home page** (`app/page.tsx`) — hero + 6 feature cards + CTAs + spec landing copy. → deliverable: `/`
6. **Skill Amp** — quiz engine (`QuizStepper` + `QuestionCard`), scoring across 6 skills, persona assignment, result page with recharts radar + persona badge + recommendations. → deliverable: full flow `/skill-amp` → `/skill-amp/quiz` → `/skill-amp/result`, result persisted to `skill_amp_results`
7. **Daily Req** — today's 3–5 questions, instant per-question feedback, persist completion. → deliverable: `/daily-req`
8. **Req Gym** — reuse `QuizStepper`, knowledge-area picker, attempts persisted. → deliverable: `/req-gym`
9. **Templates** — 5 cards, download from Supabase Storage, increment `template_downloads`. → deliverable: `/templates`
10. **Feedback form** — rating widget + `used_for_real_work` toggle + comment; persist to `feedback`. → deliverable: `/feedback`
11. **Content authoring** (long pole) — 30–40 Skill Amp Qs, 10 Daily Req Qs, 30 Req Gym Qs, 5 template files. → deliverable: seed data + template files in repo
12. **QA** — responsive (mobile/desktop), a11y (keyboard nav, ARIA, focus traps in dialogs), Lighthouse, empty/error/loading states. → deliverable: passing checks
13. **Deploy + Beta Jam launch** to 20–30 internal users. → deliverable: live URL

## Risks and Mitigation

| Risk | Mitigation |
|------|------------|
| Content is the long pole (quiz banks + 5 template files) | Treat content as parallel workstream; start authoring early; consider placeholder content for first dev iteration |
| Req Gym accuracy / IIBA credibility | Engage a BABOK-knowledgeable reviewer; use "IIBA-aligned practice" wording (never "official exam") |
| Persona rule ambiguity | Finalize mapping table with user before Skill Amp build (sign-off requested in plan) |
| Auth scope creep (SSO vs magic-link) | Lock to magic-link + email-domain allowlist for v0.1; SSO deferred unless internal policy requires |
| Template file format undecided (DOCX/MD/both) | Default to DOCX (most useful to users); confirm with user |
| Supabase data residency / internal policy | Confirm with user before provisioning project |
| Internal-only access leak | RLS on every table + allowlist at sign-up + no public anon writes |
